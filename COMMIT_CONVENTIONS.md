# Соглашения по коммитам — keypartner_app

## Префикс и формат

Все коммиты используют префикс **ВКР-NNN** с порядковым номером.

```
ВКР-NNN: Краткое описание (до 72 символов)

- Деталь 1
- Деталь 2
- Деталь 3
```

## Нумерация

Номер строго возрастающий. Следующий свободный: **ВКР-015**

| Диапазон | Тип работ |
|----------|-----------|
| ВКР-001–003 | Документы, заявления, README |
| ВКР-004–007 | Анализ 1-й недели (ИТ-инфраструктура, требования, AS IS) |
| ВКР-008–013 | Работы 2-й недели (документы, планирование) |
| ВКР-014 | Модель TO BE — UML Activity Diagram |
| ВКР-015–... | Работы 2-й недели и далее |

## Правила

1. **Один коммит = одна логическая единица работы.** Не смешивать разные задачи.
2. **Коммитить всё наблюдаемое** — не только код, но и документы (.md, .docx), диаграммы (.drawio, .png), SQL-скрипты, макеты.
3. **Тело коммита** — 3–4 пункта через `-`, что конкретно сделано.
4. **Язык** — русский.
5. **Changelog в README** — обновлять раздел `## Changelog` при завершении значимого этапа (конец недели, крупная фича). Не после каждого коммита — только вехи: что стало готово по итогу. Статусы: ✅ готово / 🔄 в работе / 📅 запланировано.
6. **Цель: ≥50 коммитов** к 07.06.2026. Сейчас: ~16. Осталось: ~34.
