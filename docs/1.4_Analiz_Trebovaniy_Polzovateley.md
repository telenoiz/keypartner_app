# 1.4 Анализ требований пользователей к веб-сервису

## 1.4.1 Категории пользователей системы

Веб-сервис для централизованного управления заявками предполагает три роли пользователей:

| Роль | Описание |
|------|----------|
| **Администратор** | Сотрудник, ответственный за техническое сопровождение системы, управление учётными записями и настройку справочников |
| **Менеджер (Сотрудник)** | Сотрудник компании, осуществляющий приём, обработку и контроль исполнения заявок |
| **Клиент (Пользователь)** | Внешний пользователь — физическое или юридическое лицо, подающее заявки на услуги ООО «Ки Партнер ИТ» |

## 1.4.2 Метод сбора требований

Требования пользователей сформированы методом анализа текущего бизнес-процесса (AS IS), изучения аналогичных систем управления заявками, а также на основе функциональных потребностей каждой категории пользователей.

## 1.4.3 Формализованные требования пользователей

| ID | Роль | Требование пользователя | Приоритет | Обоснование |
|----|------|------------------------|-----------|-------------|
| F01 | Клиент | Регистрация в системе и авторизация по логину/паролю | Высокий | Необходима идентификация для доступа к личному кабинету |
| F02 | Клиент | Создание новой заявки с указанием темы, описания и вложения файлов | Высокий | Основная функция взаимодействия клиента с компанией |
| F03 | Клиент | Просмотр актуального статуса своих заявок в личном кабинете | Высокий | Клиент должен знать о прогрессе без звонков менеджеру |
| F04 | Клиент | Получение уведомлений (в системе) при изменении статуса заявки | Средний | Снижает нагрузку на менеджера, повышает качество сервиса |
| F05 | Клиент | Просмотр истории всех ранее поданных заявок и переписки по ним | Средний | Возможность отслеживать архив обращений |
| F06 | Менеджер | Просмотр списка всех входящих заявок с фильтрацией по статусу, дате, клиенту | Высокий | Оперативный контроль очереди заявок |
| F07 | Менеджер | Обработка заявки: изменение статуса, добавление комментариев, прикрепление файлов | Высокий | Основная рабочая функция менеджера |
| F08 | Менеджер | Назначение заявки конкретному исполнителю из числа сотрудников | Высокий | Устраняет устную передачу задач |
| F09 | Менеджер | Переписка с клиентом внутри карточки заявки | Средний | Единая история коммуникаций по заявке |
| F10 | Менеджер | Выгрузка отчёта по заявкам в форматах XLSX и DOCX | Средний | Формирование управленческой отчётности |
| F11 | Администратор | Создание, редактирование и блокировка учётных записей пользователей | Высокий | Управление доступом к системе |
| F12 | Администратор | Назначение и изменение ролей пользователей (Клиент / Менеджер / Администратор) | Высокий | Разграничение прав доступа |
| F13 | Администратор | Просмотр сводной статистики: количество заявок по статусам, исполнителям, периодам | Средний | Основа для управленческих решений |
| F14 | Администратор | Управление справочниками: категории заявок, статусы, приоритеты | Низкий | Гибкая настройка системы под нужды компании |

## 1.4.6 Связь требований пользователей с функциями системы

Выявленные требования F01–F14 являются основой для разработки Технического задания (Раздел 1.7). Каждое требование пользователя трансформируется в одну или несколько функциональных возможностей системы (функциональные требования к АС), описанных в ТЗ по ГОСТ 34.602-2020.

| Требование | Раздел ТЗ |
|-----------|-----------|
| F01, F11, F12 | 4.1 Требования к функциям: управление доступом |
| F02, F06, F07, F08 | 4.1 Требования к функциям: управление заявками |
| F03, F04, F05, F09 | 4.1 Требования к функциям: уведомления и история |
| F10, F13 | 4.1 Требования к функциям: отчётность |
| F14 | 4.1 Требования к функциям: администрирование справочников |
