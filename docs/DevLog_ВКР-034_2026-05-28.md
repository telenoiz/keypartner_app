## PHASE 1 — Анализ (результаты)

**Release Notes:** текущая версия 0.4 (28.05.2026). ВКР-034 → Release 0.5.

**Проблемы найдены:**

1. В `core/urls.py` нет маршрутов `/services/`, `/news/`, `/projects/` (Critical)
2. В `core/views.py` нет view-функций для трёх страниц (Critical)

4. Лимиты списков без именованных констант — потенциальный hardcode (Critical — превентивно)
5. На `home.html` нет ссылки «Все услуги →» → dead end (High)
6. `base.html` навигация не содержит контентных страниц (High)
7. `Project` не имеет `PORTFOLIO_STATUSES` — нет именованного списка статусов (High)
8. CSS: нет компонентов для news, projects, page-hero, badge (High)
9. `base.html` оборачивает content в `.container` — новые шаблоны не дублируют (High)

### Ключевые решения

**`models.py`:**
- `PORTFOLIO_STATUSES` — class-level constant, не inline-список в view

**`views.py`:**
- `select_related('category')` в services_view — избегаем N+1 запросов
- `select_related('service', 'service__category')` в projects_view
- Лимиты через именованные константы, нет `[:3]` магических литералов

**Шаблоны:**
- Нет inner `.container` — учтён баг base.html (Tech Debt, отдельный коммит)
- `project.user` не выводится — защита личных данных
- Все `<time>` теги с атрибутом `datetime` — семантический HTML
- `aria-labelledby` на каждой `<section>`
- Empty state через `role="status"`

**CSS:**
- `--badge-*` переменные добавлены в `:root` из таблицы 2.1_Color_Typography.md
- Нет ни одного raw HEX в новых компонентах (ВКР-034 блоки)
