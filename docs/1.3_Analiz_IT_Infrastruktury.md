# 1.3 Анализ ИТ-инфраструктуры ООО «Ки Партнер ИТ»

## 1.3.1 Общая характеристика инфраструктуры

ООО «Ки Партнер ИТ» — организация численностью до 20 сотрудников, специализирующаяся на разработке и внедрении программных решений для корпоративных заказчиков. Деятельность компании предполагает постоянное взаимодействие с клиентами: приём заявок, согласование технических требований, передача задач исполнителям и контроль результатов.

## 1.3.2 Аппаратное обеспечение

| Компонент | Характеристика |
|-----------|---------------|
| Рабочие станции | 10–15 единиц (ноутбуки и настольные ПК) |
| Процессор | Intel Core i5/i7, 8–16 ГБ ОЗУ |
| Хранение данных | Локальные диски + облачное хранилище (Яндекс.Диск) |
| Сетевое оборудование | Маршрутизатор, коммутатор, Wi-Fi точки доступа |
| Периферия | Принтеры, сканеры |

## 1.3.3 Программное обеспечение

| Категория | Используемые решения |
|-----------|---------------------|
| Операционная система | Windows 10 / Windows 11 (64-бит) |
| Офисный пакет | Microsoft Office 2019 / Office 365 |
| Учёт заявок | Microsoft Excel (ручная таблица) |
| Коммуникации | Электронная почта (Яндекс.Почта), Telegram |
| Разработка | Visual Studio Code, Python, Git |
| Браузеры | Google Chrome, Mozilla Firefox |
| Антивирус | Kaspersky Endpoint Security |

## 1.3.4 Сетевая инфраструктура

Локальная сеть организации построена на базе технологии Ethernet (100/1000 Мбит/с) с использованием Wi-Fi (стандарт 802.11ac). Выход в Интернет осуществляется через выделенный канал провайдера. Совместный доступ к файлам обеспечивается через облачное хранилище. Единый корпоративный сервер отсутствует.

## 1.3.5 Текущая схема взаимодействия с клиентами (AS IS)

На момент начала разработки автоматизированной системы взаимодействие с клиентами и учёт заявок ведётся следующим образом:

1. **Приём заявок** — через телефонные звонки, электронную почту и мессенджеры (Telegram). Единая точка входа отсутствует.
2. **Фиксация** — менеджер вручную вносит данные о заявке в Excel-таблицу. Файл хранится локально или в облаке.
3. **Назначение исполнителя** — устно или через сообщение в мессенджере. Статус отслеживается вручную.
4. **Уведомление клиента** — по завершении работ менеджер самостоятельно связывается с клиентом (звонок / email).

## 1.3.6 Выявленные проблемы и ограничения

| # | Проблема | Последствие |
|---|----------|-------------|
| 1 | Отсутствие единой точки входа для заявок | Потеря обращений, дублирование |
| 2 | Ручной учёт в Excel-таблицах | Ошибки, отсутствие актуального статуса |
| 3 | Нет автоматического уведомления клиентов | Низкий уровень клиентского сервиса |
| 4 | Отсутствие разграничения ролей и прав доступа | Риски безопасности данных |
| 5 | Невозможность формировать аналитику и отчёты | Нет данных для управленческих решений |
| 6 | Нет истории переписки и изменений по заявке | Трудности при разрешении спорных ситуаций |

## 1.3.7 Выводы

Текущая ИТ-инфраструктура обеспечивает базовые потребности компании, однако не поддерживает эффективное управление клиентскими заявками. Рабочие места готовы к развёртыванию веб-сервиса — наличие современных браузеров и стабильного интернет-канала позволяет использовать веб-интерфейс без установки дополнительного клиентского ПО. Разрабатываемый веб-сервис устранит выявленные ограничения и автоматизирует процесс взаимодействия с клиентами.
